# 1 함수와 변수

**코틀린에서 함수를 선언하는 방법**

## 1.1 일반적인 선언 방법
```
fun sum(a: Int, b: Int):Int {
    return a + b
}
```
함수 선언은 fun 키워드를 사용한다.  그리고 함수 이름을 지정해주고  
괄호 안의 파라미터는 ``파라미터명 : 타입`` 의 형태를 가진다  
함수의 반환 타입은 파라미터 목록의 닫는 괄호 뒤에 ``: 반환 타입`` 형태로 적어주게 된다  
값을 반환하지 않는 함수의 경우에는 반환 타입을 명시할 필요가 없다  

### 1.1.1 표현식이 본문인 함수
```
fun sum(a: Int, b: Int) = a + b
```
함수명과 파라미터까지는 일반적인 선언 방법과 다르지않으나 중괄호가 없이 ``= a + b``가 들어가있다  
이는 함수를 좀 더 간결하게 표현하는 방법으로 a + b라는 표현식의 결과값이 반환된다  
이 경우 컴파일러가 a + b에서 타입을 추론할 수 있으므로 반환값을 명시하지 않아도 된다

## 1.2 변수

**코틀린에서 변수를 선언하는 문법**
```
val a: Int = 1
val b = 2
var c: int = 3
var d = 4
```
변수 선언은 val과 var로 시작하며, 이어서 ``변수명 : 타입``의 순서로 이루어진다  
다른 언어들과 마찬가지로 선언과 동시에 값을 초기화 할 수 있으며  
타입을 생략할 수 있지만 값을 초기화하지 않는 경우 아래와 같이 타입을 명시해줘야 한다  
```
val a: Int
a = 1
```

### 1.2.1 가변 변수와 불변 변수  
변수를 선언하는 키워드는 ``var`` 과 ``val`` 이 있는것을 알았다  
``val``의 경우 한 번 초기화 되면 이후 변경이 불가능한 값을 나타내고  
``var``의 경우 초기화 이후에도 계속 값을 변경할 수 있다
```
val a = 1
a = 2 // 컴파일 에러가 뜬다
var b = 3
b = 4 // 정상적으로 실행된다
```
# 2 흐름 제어 
코틀린의 흐름 제어는 자바와 유사하지만 큰 차이가 있다  

## 2.1 if - else 문
```
val dollar = 4
val class: String

if(dollar >= 4) {
    class = "부자"
}else {
    class = "안부자"
}
```
코틀린의 if - else 문은 각 분기의 마지막 표현식의 결과값을 반환한다는 점에서  
자바의 if - else 문과 다르다  따라서 아래와 같이 변경이 가능하다  
```
val dollar = 4
val class = if(dollar >= 4) {
    "부자"
} else {
    "안부자"
}
```
if - else는 분기 내에 표현식이 하나뿐이라면 중괄호를 생략 가능하다  
이 특성을 사용하면 아래와 같이 3항 연산자를 대체할 수 있다  
```
val dollar = 4
val class = if(dollar > 4) "부자" else "안부자"
```
